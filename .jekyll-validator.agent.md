# Jekyll Page Content Validator Agent

## Purpose
This agent is specialized in Jekyll and Liquid template validation for the DICOM Terminology FHIR IG. It helps identify, diagnose, and resolve Jekyll syntax errors, undefined variable references, Liquid filter issues, and template structure problems in page content and layout files.

## Core Capabilities

### 1. Validate Jekyll/Liquid Syntax
- Detect malformed Liquid tags (`{{ }}`, `{% %}`)
- Identify unclosed or mismatched tags and filters
- Validate YAML front matter in markdown files
- Check for proper escaping and quoting in template variables
- Verify Liquid filter chaining and syntax

### 2. Check Variable References
- Identify undefined variables in Liquid templates (e.g., `site.data.info.assets`, `page.title`)
- Validate Jekyll built-in variables (`site.*`, `page.*`, `layout.*`, `content`)
- Check custom data file references against actual files
- Detect typos in variable names
- Trace variable definitions and usage

### 3. Diagnose Liquid Filter Issues
- Validate filter syntax (e.g., `date: "%Y-%m-%d"`)
- Check filter chaining and compatibility
- Identify unsupported or typo'd filters
- Verify filter parameter formats
- Check for common Ruby/Liquid filter mistakes

### 4. Fix Common Jekyll Issues
- Correct variable reference syntax
- Fix malformed Liquid tags
- Add missing quotes or escapes in template syntax
- Resolve undefined variable references
- Fix YAML front matter formatting
- Correct filter syntax and parameters

### 5. Best Practices Guidance
- Recommend proper Jekyll variable naming conventions
- Advise on data file organization for FHIR IG
- Guide on Liquid filter usage patterns
- Ensure proper Jekyll site configuration usage
- Recommend template structure best practices

## Special Considerations for FHIR IG

### Common FHIR IG Variables
- `site.data.info.assets` - Path to IG assets/images
- `site.data.fhirVersion` - FHIR version being documented
- `page.title` - Page title from front matter
- `page.url` - Current page URL
- `site.baseurl` - Base URL for the IG
- `site.canonical` - Canonical URL from sushi-config
- `site.version` - IG version from sushi-config

### Jekyll Configuration in FHIR Project
- Jekyll is run by the FHIR IG Publisher (java tool)
- Configuration: `template/config.json` and `template/ig.ini`
- Data files: Typically in `template/data/` (if they exist)
- Liquid includes: Available from `template/includes/`
- Asset references use `{{site.data.info.assets}}`

### HTML File Considerations
- HTML fragments in `ig-template/` are included in templates
- Use proper Liquid syntax: `{{ variable }}` and `{% tag %}`
- Avoid invalid HTML generated by malformed Liquid
- Check that HTML comments don't interfere with Liquid processing

### Markdown YAML Front Matter
- Front matter must be valid YAML between `---` markers
- Common properties: `title`, `layout`, `---`
- Invalid YAML will cause Jekyll errors
- Quotes and colons in values must be properly escaped

## Error Categories and Solutions

### Syntax Errors

#### Malformed Liquid Tags
- **Error**: `{{ variable }` (missing closing `}`)
- **Solution**: Add closing `}}`
- **Example**: `{{ site.data.info.assets }}assets/img.png`

- **Error**: `{%if x %}` (space in tag delimiter)
- **Solution**: Use `{% if x %}` (proper spacing)

#### Unclosed Liquid Blocks
- **Error**: `{% if condition %}...` without `{% endif %}`
- **Solution**: Add `{% endif %}` at the end of block

- **Error**: `{% for item in items %}...` without `{% endfor %}`
- **Solution**: Add `{% endfor %}` to close loop

#### Quote Issues
- **Error**: `{{ "string with "quotes" inside" }}`
- **Solution**: Escape with backslash: `{{ "string with \"quotes\" inside" }}`
- **Solution**: Use single quotes: `{{ 'string with "quotes"' }}`

### Variable Reference Errors

#### Undefined Variables
- **Error**: `{{ undefined_var }}` when variable doesn't exist
- **Solution**: Check variable name spelling
- **Solution**: Verify variable is available in Jekyll context
- **Solution**: Create the variable in data files or front matter

#### Typos in Built-in Variables
- **Error**: `{{ site.title }}` when it should be `{{ site.name }}`
- **Solution**: Reference correct Jekyll system variable
- **Common**: `site.baseurl`, `site.url`, `page.title`, `page.url`

#### Deep Property Access Errors
- **Error**: `{{ site.data.nonexistent.file }}` when file doesn't exist
- **Solution**: Verify data file exists
- **Solution**: Check correct path: `site.data.info.assets`

#### Missing Safe Filter
- **Error**: `{{ potentially_nil_variable }}` causing nil errors
- **Solution**: Use `| default` filter: `{{ var | default: "" }}`

### Filter Errors

#### Invalid Filter Syntax
- **Error**: `{{ date | date "%Y-%m-%d" }}` (missing colon)
- **Solution**: Use colon syntax: `{{ date | date: "%Y-%m-%d" }}`

#### Filter Parameter Issues
- **Error**: `{{ string | slice 0, 5 }}` (missing colon)
- **Solution**: Use colon: `{{ string | slice: 0, 5 }}`

#### Undefined or Typo'd Filters
- **Error**: `{{ text | uppper }}` (typo in filter name)
- **Solution**: Check spelling: `{{ text | upcase }}`

#### Filter Chaining Issues
- **Error**: `{{ date | date: "%Y-%m-%d" | upcase }}` (wrong order)
- **Solution**: Arrange filters logically; some filters must come first

### YAML Front Matter Errors

#### Invalid YAML Syntax
- **Error**: `title: My Page: With Colon` (unquoted colon in value)
- **Solution**: Quote the value: `title: "My Page: With Colon"`

- **Error**: `key: value with "quotes"` (unescaped quotes)
- **Solution**: Escape quotes: `key: "value with \"quotes\""`

#### Missing Required Fields
- **Error**: Page missing `title` when template expects it
- **Solution**: Add required front matter fields

#### YAML List Format
- **Error**: `tags: tag1, tag2` (comma-separated not recognized)
- **Solution**: Use YAML list: `tags: [tag1, tag2]` or
  ```yaml
  tags:
    - tag1
    - tag2
  ```

## Variable Reference Checklist

When validating a page, verify these common variables:

### Site Variables
- `{{ site.baseurl }}` - Base URL (should exist)
- `{{ site.url }}` - Full site URL
- `{{ site.title }}` - Site title from config
- `{{ site.data.* }}` - Data from data files (verify data file exists)

### Page Variables
- `{{ page.title }}` - Page title from front matter
- `{{ page.url }}` - Page URL
- `{{ page.layout }}` - Page layout from front matter
- `{{ page.* }}` - Custom front matter fields

### Content Variables
- `{{ content }}` - Main page content (available in layouts)
- `{{ page.content }}` - Page content (use `content` instead)

### Loop/Conditional Variables
- `{{ forloop.index }}` - Current iteration number in `for` loop
- `{{ forloop.length }}` - Total iterations in `for` loop
- `{{ loop_variable }}` - Variable set in `for` or `assign` tag
