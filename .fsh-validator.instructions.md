## Understanding FSH Syntax

### Key FSH Constructs for DICOM Project
```fsh
// CodeSystem definition
CodeSystem: DICOM-DCM-Codes
Id: dicom-dcm-codesystem
Title: "DICOM Codes"
Description: "DICOM concept codes"
* #100000 "Display Text"
  """Longer description in triple quotes"""
  * ^property[0].code = #keyword
  * ^property[0].valueString = "KeywordName"
  * ^property[1].code = #retired
  * ^property[1].valueBoolean = false

// Common mistakes:
// - Missing quotes around display text
// - Incorrect indentation for properties
// - Wrong property names
// - Invalid boolean values
// - Duplicate code definitions
// - Malformed descriptions
```

### Property Syntax for DICOM
The DICOM CodeSystems use a consistent pattern:
- `keyword` property: CamelCase human-readable name
- `retired` property: Boolean (false/true or "false"/"true")
- Additional properties as needed

## Error Diagnosis Guide

### Error Type: "Property name not recognized"
**Cause**: Misspelled property name or invalid FSH syntax
**Solution**: Check FSH spec reference for correct property name and syntax

### Error Type: "Invalid string format"
**Cause**: Missing quotes, unclosed strings, or special character issues
**Solution**: Wrap strings in double quotes, escape special characters

### Error Type: "Duplicate element"
**Cause**: Same code defined twice in the same CodeSystem
**Solution**: Remove duplicate or rename if intentional

### Error Type: "Invalid cardinality"
**Cause**: Incorrect cardinality syntax (e.g., [0..2] instead of [0..1])
**Solution**: Use correct FHIR cardinality: [0..0], [0..1], [0..*], [1..1], [1..*]

### Error Type: "Resource not found"
**Cause**: Referenced ValueSet or CodeSystem doesn't exist
**Solution**: Check filename and path, ensure resource is defined

### Error Type: "Invalid value type"
**Cause**: Property expects boolean but got string, or number instead of string
**Solution**: Convert value to correct type (true/false for boolean, "text" for string)

## Common DICOM-Specific Issues

### Large CodeSystem (21000+ codes)
- The main DCM CodeSystem file is large; compilation may take time
- Be patient with SUSHI compiler
- Consider running on reduced subsets if needed for testing

### Code Format
- DICOM codes are typically numeric (e.g., #100000, #113994)
- Display text is in English
- Description can contain longer explanatory text

### Property Definitions
- DICOM CodeSystems should have `keyword` and `retired` properties defined
- Property definitions follow a strict pattern - don't improvise
